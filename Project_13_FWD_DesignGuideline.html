Design Guideline - FWD Cube

 

Project Brief

This project delivers comprehensive design guidelines for an all-in-one super App. It provides clear, actionable standards covering UI details, visual effects, interaction behaviors, real-world usage cases, module components, specific user flows, and exceptional scenarios. The design guidelines establish consistent yet flexible standards that enable fast, high-quality implementation across platforms. They ensure every interface element feels intuitive and culturally appropriate, reducing cognitive load for field agents and improving overall adoption and efficiency.

Target Users

Designers of all ages that is referring to the CUBE Design guideline for their daily tasks

My Role

  • Design guideline creator & manager
  • Researcher

Main Activities

  • Create guidelines when new components are designed
  • Manage the design guideline library
  • Test the guidelines with real users
  • Iterate on the guidelines based on user feedback
  • Research best practice of existing UX behavior for enhancing usability of the components/patterns

Project Achievements

Showcase - Overlay

Existing problem

  1. We usually encounter issues on when to use popups, tooltips, dialogs, bottom sheets and other overlay components.
  2. There is no clear definitions for those overlay related components.
  3. There is no clear guidelines for overlay usage.
  4. Inconsistent overlay behavior across different platforms, leading to confusion for field agents.
  5. Lack of clear guidelines on when and how to use overlays effectively.

Research

I did some research online for the definition of each overlay components.

Summarize the definitions and usage guidelines

Well defined all types of overlay components as table of content

I created the diagram of the relations for different overlay components, including the definition, usage guidelines, real cases and exceptional scenarios. So everytime when designers get confused about which overlay component to use, they can refer to the diagram and find the answer.
